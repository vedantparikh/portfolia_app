üéØ Project Overview

Vision Statement

Build a comprehensive, intelligent portfolio management application that provides real-time tracking, analysis, and automated insights for diverse investment portfolios across multiple asset classes and currencies.

Core Features

  • Multi-Asset Portfolio Tracking: Stocks, bonds, commodities, ETFs, mutual funds
  • Multi-Currency Support: USD, EUR, GBP, JPY, and other major currencies
  • PDF Transaction Extraction: Automated parsing of investment statements and reports
  • Advanced Analytics: Performance metrics, risk analysis, and benchmarking
  • Automated Reporting: Daily summaries via email/WhatsApp with PDF generation
  • Trading Algorithms: AI-powered buy/sell signals for stocks
  • Interactive Dashboards: Real-time portfolio visualization and insights

Target Users

  • Individual Investors: Retail investors managing personal portfolios
  • Financial Advisors: Professionals managing client portfolios
  • Institutional Investors: Fund managers and investment firms
  • Financial Analysts: Professionals requiring detailed portfolio analytics

High-Level Architecture

üé® System Overview

The application follows a microservices architecture with event-driven communication, ensuring scalability, maintainability, and high availability.

üè¢ Architecture Layers:
  • Presentation Layer: React.js frontend with responsive design
  • API Gateway: FastAPI-based routing and authentication
  • Business Logic Layer: Microservices for different domains
  • Data Layer: PostgreSQL, Redis, and file storage
  • Infrastructure Layer: Docker, Kubernetes, and cloud services

Microservices Architecture

üîÑ Event-Driven Communication

Services communicate through Apache Kafka for asynchronous, reliable message passing. This ensures loose coupling and high scalability.

Database Architecture

üìä Multi-Database Strategy

  • PostgreSQL: Primary relational database for transactional data
  • Redis: In-memory caching and session management
  • MongoDB: Document storage for unstructured data
  • TimescaleDB: Time-series data for market prices and analytics

Data Relationships

üîó Entity Relationship Diagram

The database design follows a normalized structure with proper foreign key relationships, ensuring data integrity and efficient querying.

Key Relationships:
  • User ‚Üí Portfolio: One-to-many relationship
  • Portfolio ‚Üí Asset: Many-to-many through portfolio_assets
  • Asset ‚Üí Price: One-to-many for historical data
  • Transaction ‚Üí Portfolio: Many-to-one relationship
